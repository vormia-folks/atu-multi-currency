# Release notes — v2.1.0

**Release date:** 2026-05-11  

**Full tag:** `v2.1.0`  

Structured changelog: [`CHANGELOG.md`](../../CHANGELOG.md).

This minor release tightens package wiring, improves operator-facing CLI output, and refines the base-currency seeder. It builds on the **v2.0.0** model where migrations, API routes, merged config, and Volt admin UI load from `vendor` rather than being copied into the application.

## Summary

- **Service provider & paths** — `ATUMultiCurrency` exposes `migrationsPath()` and `migrationsPathRelativeToBase()` so tooling (for example `atumulticurrency:refresh`) can resolve package migration files reliably when rolling back with Artisan `--path`.
- **Boot wiring** — Migrations and routes continue to register from the package; behavior is clarified in console output and docs.
- **`atumulticurrency:help`** — Streamlined summary: commands, options, env keys, HTTP prefixes/names, seeder class, and optional config publish tag.
- **`atumulticurrency:ui-install`** — Stricter dependency checks (including `vormiaphp/ui-livewireflux-admin`), clearer guidance when Volt or Flux is missing, and improved optional Flux sidebar injection messaging.
- **`ATUMultiCurrencySeeder`** — Logging and console feedback go through small `info` / `warn` helpers (command output when present, plus structured `Log` lines) for easier support and log aggregation.

## Upgrade notes

From **v2.0.0**, upgrade with Composer only; run migrations if new migration files ship in a future patch (none are required solely for this documentation and CLI release).

```sh
composer update vormia-folks/atu-multi-currency
php artisan atumulticurrency:ui-update
```

If you rely on `ATUMultiCurrency::VERSION` or `app('atumulticurrency.version')`, it is now **2.1.0**.
